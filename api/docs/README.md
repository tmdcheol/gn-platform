# 문서 (Spec-Driven Development)

gn-platform api는 **명세 주도 개발(Spec-Driven Development)** 방식으로 작업한다.
기능마다 `spec → plan → tasks` 순서로 문서를 작성하고, 문서를 기준으로 구현한다.

## 3단계 문서

| 문서 | 질문 | 내용 |
|------|------|------|
| `spec.md` | 무엇을 / 왜 (What/Why) | 기능 명세: 요구사항, 도메인 모델, 수용 기준 |
| `plan.md` | 어떻게 (How) | 개발 계획: 설계, 파일 구조, 기술 선택 근거 |
| `tasks.md` | 순서 (Steps) | 구현·테스트 체크리스트. `- [ ]` / `- [x]` 로 진행 추적 |

세 파일은 `api/` 바로 아래에 두고, 현재 진행 중인 기능 기준으로 갱신한다.

## 진행 방식

1. `spec.md` 작성 → 요구사항 확정 (모호함 제거)
2. `plan.md` 작성 → 설계 및 리뷰
3. `tasks.md` 분해 → 실행 가능한 작업 목록으로 쪼갬
4. task 하나씩 구현 → 완료 시 `- [x]` 체크 후 커밋
5. 요구가 바뀌면 `spec.md`부터 갱신 → 문서와 코드를 동기화 유지

## 구조

```text
api/
├── spec.md     # 기능 명세
├── plan.md     # 개발 계획
├── tasks.md    # 구현·테스트 리스트
└── docs/
    └── README.md   # 이 문서 (작업 방식 안내)
```
